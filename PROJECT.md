Спецификация программы SvedUch:

1. Назначение программы

SvedUch — это Python-приложение, которое позволяет формировать базу данных со сведениями об учениках школы. Пользователь вводит общие сведения об учениках, информацию о заключении ПМПК по конкретному ученику, о рекомендованной для него программе обучения и рекомендациях об учебном процессе данного ученика для конкретных специалистов школы, о документе, издаваемой школой при переводе такого ученика на конкретную программу обучения. Информация хранится в общей таблице. Программа позволяет производить выборку в окне "Выборки" информации об учениках по критериям: номер класса, рекомендованная программа; результаты выборки показывает в виде временной таблицы. В программе имеется вкладка "Перевод", в которой можно выбрать учеников конкретного класса, отметить конкретного ученика или всех вместе, проставить причину перевода и осуществить перевод. Кнопка "Сохранить" для внесения сведений в основную таблицу БД.

2. Стек

- Python 3.11+
- SQLite 
- Библиотека для запросов на твое усмотрение
- GUI PyQt 

3. Таблицы базы данных

Программа хранит следующие данные в базе данных SQLite:
- Классы (forms) - таблица классов школы. Поля: номер. В состав номера входят цифры и буквы
- Программы (programs) - таблица программ. Поля: наименование, версия
- Рекомендации (recommendations) - таблица рекомендаций специалистам. Поля: наименование специалиста, наименование рекомендации.
Одному специалисту могут быть даны несколько рекомендаций
Если будет несколько рекомендаций одному специалисту (например 4), то в таблице будет 4 строки с этим специалистом, но с разными рекомендациями
- Ученики (pupils) - таблица учеников. Поля: номер класса, фамилия, имя, отчество, дата рождения, дата заключения ПМПК, номер заключения ПМПК, ссылка на программу (id из таблицы programs — для однозначной группировки при выборках по количеству учеников на программе), номер приказа, дата приказа, рекомендации (количество полей равно количеству специалистов)
Предполагается, что будет 5 специалистов, которым даются рекомендации. Включать в таблицу нужно всех специалистов, при этом если специалисту дано несколько рекомендаций, то они вносятся через точку с запятой, если рекомендаций не дано, то вносится слово "нет"
- Архив (pupils-history) - таблица с архивными сведениями об учениках, формируется при переходе ученика в другую школу или при окончании школы. Поля анологичны таблице pupils, добавлены 2 поля - "дата перевода" и "причина перевода"  
- Настройки (settings) - хранение настроек программы. Поля: ключ (key), значение (value). Примеры: путь к файлу БД, тема интерфейса, размер и положение главного окна. Реализация в виде таблицы «ключ — значение» (одна строка на параметр).


4. Рабочий процесс

4.1 Интерфейс программы

На экране пользователь видит кнопки для открытия окон: Таблицы, Выборки, Перевод
В окне "Таблицы" должны быть кнопки для открытия всех таблиц

4.2 Ввод сведений

Пользователь:

- вводит сведения о классах школы, о программах, о специалистах и рекомендациях
- вводит сведения об учениках: ФИО, дата рождения, сведения о заключении ПМПК, информацию о приказе школы. Остальная информация заносится на основе выпадающего меню со сведениями соответствующей таблицы
При внесении сведений об ученике результаты формируются в виде временной таблицы (в памяти, не в SQLite), после нажатия кнопки "Сохранить" сведения заносятся в таблицу pupils, временная таблица очищается

4.3 Интерфейс результатов

При вводе сведений об ученике на экране пользователь видит окна: 1-й ряд - "Класс", "Фамилия", "Имя", "Отчество", "Дата рождения"; 2-й ряд - "Дата заключения ПМПК", "Номер заключения ПМПК", "Номер приказа", "Дата приказа"; 3-й ряд - "Программа", "Версия"; ниже расположен блок "Рекомендации специалистам"
У окна "Класс" иметь значок для открытия всплывающего меню, в котором отражаются номера классов школы из таблицы "forms", после нажатия на номер нужного класса меню закрывается, а окно заполняется
У окна "Программа" иметь значок для открытия всплывающего меню с информацией таблицы "Programs", после выбора конкретной программы меню закрывается, наименование вносится в окно "Программа" и версия программы в окно "Версия"
В блоке рекомендаций расположить ряд из пяти кнопок с наименованиями специалистов из таблицы "recommendations", ниже кнопок иметь ряд окон, в которых будут отражаться рекомендации для конкретного специалиста из таблицы "recommendations"
У каждого окна специалиста иметь значок для открытия всплывающего меню с информацией о назначенных ему рекомендациях из таблицы "recommendations", предусмотреть возможность выбора нескольких рекомендаций одновременно и отражения их в окне ниже кнопки через точку с запятой

4.4 Сохранение введенной информации

Когда пользователь нажимает кнопку "Сохранить":

- информация заносится в таблицу pupils
- окна ввода очищаются.

4.5 Формирование запросов

В окне "Выборки" предусмотреть возможность формирования запроса на извлечение из таблицы pupils сведений об учениках конкретного класса, об учениках с конкретной программой, а также выборку по количеству учеников, находящихся на определённой программе (агрегация по программе: программа — количество учеников). Связь pupils с programs через id программы обеспечивает корректную группировку при таких запросах.
Информация формируется в виде временной таблицы (в памяти, не в SQLite), иметь возможность вывода как всех полей таблицы pupils, так и выборки только необходимых полей. Предусмотреть возможность выгрузки результатов запроса в виде файла в формате "Excel", иметь кнопку "Очистить" для удаления результатов запроса.


4.6 Перевод учеников

В окне "Перевод" проводится работа по изменению сведений об ученике при переходе в другую школу, в другой класс, на другую программу.
В окне отображаются 2 блока: перевод ученика и перевод класса.
В блоке "Перевод класса" производится выбор учеников по номеру класса в виде временной таблицы (в памяти, не в SQLite), имеется два поля для ввода даты и причины перевода, а также кнопка "Сохранить"
В блоке "Перевод ученика" производится выбор конкретного ученика по Фамилии, Имени, Отчеству и номеру класса в виде временной таблицы (в памяти, не в SQLite), имеется два поля для ввода даты и причины перевода, а также кнопка "Сохранить"
При переводе ученика в другую школу информация о нем переносится в таблицу pupils-history, в других случаях изменяются записи в pupils
Перевод класса осуществляется в конце учебного года. При этом при отработке сведений по одиннадцатым классам после нажатия кнопки "Сохранить" информация переносится в таблицу pupils-history, у учеников других классов в таблице pupils автоматизированно изменяется номер класса: число увеличивается на единицу, буква остается та же
В интерфейсе явно выбирать тип перевода (например: «в другую школу» → в архив, «в другой класс/программу» → правка в pupils)

   3. Все настройки сохраняются в таблицу settings в базе данных sveduch.db

8. Добавление новых полей

В таблицы Ученики (pupils) и Архив (pupils-history) нужно добавить два новых поля - Домашний адрес и Пол. Новые поля вставить между полями Дата рождения и Дата ПМПК. Ширина поля Домашний адрес - 50 символов, поля Пол - 4 символа. Новые поля надо добавлять и в создаваемые временные таблицы.
Во вкладках Добавить ученика и Изменения по ученику раздела Таблицы - Ученики нужно добавить окна для ввода сведений о домашнем адресе и поле ученика. Поля Домашний адрес и Пол вставить в новый ряд полей между существующими рядами, которые начинаются с окон Класс и Дата заключения ПМПК.
В блоке "Отображаемые поля" раздела "Выборки" добавить окна для проставления отметки для выбора полей "Домашний адрес" (можно сократить "Дом.адр.") и "Пол".

9. Загрузка сведений из файла формата Excel

Добавить возможность загрузки общих сведений об учениках из файлов в формате Excel в таблицу Ученики (pupils). Файл в формате Excel содержит таблицу с графами: Фамилия, Имя, Отчество, Дата рождения, Домашний адрес, Пол. Сведения должны грузиться в таблицу Ученики в поля с такими же наименованиями.
Загрузку осуществлять во вкладке Список учеников раздела Таблицы - Ученики. Для этого во вкладке Список учеников создать окна: 
- Класс - для ввода номера класса, информация об учениках которого будет подгружаться в базу данных. После загрузки информации об учениках номер класса должен автоматом проставляться
  в соответствующее поле таблицы Ученики по всем записям учеников загруженного класса.
- Рядом с окном Класс расположить окно для поиска и выбора необходимого файла и кнопку "Загрузить".
После нажатия кнопки "Загрузить" информация из граф файла должна копироваться в одноименные поля таблицы Ученики, а информация из окна Класс копироваться в соответствующее поле таблицы Ученики по всем загруженным записям.

10. Мониторинг

10.1 Дополнительные таблицы

Необходимо разработать функционал для ввода дополнительных данных и проведения мониторинга развития обучающихся. Для этого нужно создать основную таблицу, в которой будут отражаться результаты анализа развития ребенка, и несколько
новых вспомогательных таблиц.
Вспомогательные таблицы:
- Специалисты (experts) - таблица специалистов, которые будут вносить результаты развития ученика по своему направлению. Поля: наименование.
- Критерии (criterions) - таблица критериев анализа развития. Поля: наименование.
- Уровни (standards) - таблица уровней развития ребенка по конкретном критерию. Поля: наименование, код.
Кнопки для выбора вспомогательных таблиц расположить во вкладке "Таблицы" в следующем порядке: "Специалисты" между "Программы" и "Рекомендации", "Критерии" и "Уровни"  - одна под другой между кнопками "Рекомендации" и "Ученики". Окна с новыми таблицами должны иметь кнопки с функционалом для CRUD-запросов.

Мониторинг будет проводиться два раза в течении учебного года: за I полугодие и за II полугодие. Учебный год начинается 1-го сентября текущего года и заканчивается 31-го мая следующего года (например, текущий учебный год начался 01.09.2025 и закончится 31.05.2026).
Информацию об учебном годе необходимо отразить в основном окне программы. 
Для этого: блок "Выберите раздел:" со всеми кнопками опустить на четверть экрана. В верхней части экрана предусмотреть место для наименования программы. Ниже расположить надпись "Учебный год:" и окно для ввода сведения в формате "ГГГГ-ГГГГ". Правее окна ввода дат расположить надпись "Период" и окно для ввода, куда будет 
вводиться информация "I полугодие" или "II полугодие" 
После ввода информации об учебном годе и периоде при каждом открытии программы сведения об учебном годе и периоде должны автоматически отражаться в данных окнах. Способ и место хранения информации об учебном годе и периоде на твое усмотрение. Информация об учебном годе и периоде должна обновляться после ввода новых сведений.

10.2 Таблица анализа.

Для отражения результатов анализа развития учеников создать таблицу "Анализ", в ней предусмотреть наличие постоянных полей и полей, которые будут добавляться при проведении анализа, т.е по одному поля в каждое полугодие. Наименование поля формируется на основании сведений, отраженных в окнах ввода "Период" и "Учебный год". Добавленные поля в дальнейшем становятся постоянными.
Таблица "Анализ" (analysis) содержит поля: постоянные - класс, фамилия, имя, отчество, специалист, критерий, добавляемые - результат (за конкретный период конкретного учебного года, например: Результат за I п. 2025-2026). Поле Результат создается при заполнении окон ввода "Период" и "Учебный год" и при обновлении информации в этих окнах.
Кнопку для выбора окна ввода сведений в таблицу "Анализ" расположить во вкладке "Таблицы" ниже кнопки "Ученики".

10.3 Интерфейс окна "Анализ".

Предполагается следующий порядок работы: 
- выбор специалиста, который оценивает развитие ученика;
- выбор ученика, развитие которого будет анализироваться;
- выбор критерия оценки;
- проставление результата оценки по этому критерию.
В соответствии с порядком работы окно должно содержать 3 блока:
- верхний блок - выбор специалиста - должно быть окно "Специалист" с выпадающим меню для выбора специалиста из таблицы "Специалисты";
- следующий блок - выбор ученика - блок содержит ряд окон - "Класс", "Фамилия", "Имя", "Отчество" и кнопку "Найти". У окна "Класс" иметь значок для открытия всплывающего меню, в котором отражаются номера классов школы из таблицы "forms", после нажатия на номер нужного класса меню закрывается, а окно заполняется. Кнопка "Найти" должна иметь функционал всплывающего окна со списком выбранного
класса, иметь возможность выбора конкретного ученика. После выбора ученика всплывающее меню закрывается, а поля Фамилия, Имя и Отчество заполняются.
- следующий блок - выбор критерия - блок содержит верхний ряд кнопок "Добавить", "Сохранить" и "Очистить". Ниже их расположено два окна: "Критерий" и "Результат" и кнопка "Перенести"
При нажатии кнопки "Добавить" открывается всплывающего окна с информацией таблицы "Критерии", после выбора конкретного критерия информация о нем переносится в окно "Критерии" и всплывающее окно закрывается. 
После заполнения окна "Результат" и нажатия кнопки "Перенести" информация переносится в сформированную в блоке временную таблицу (в памяти, не в SQLite) с полями "Критерий" и "Результат", при этом окна "Критерий" и "Результат" очищаются для добавления следующего критерия.
После выбора всех необходимых критериев, проставления по ним результатов и нажатия кнопки "Сохранить" сведения заносятся в таблицу "Анализ", при этом количество записей в таблицу "Анализ" соответствует числу записей во временной таблице, поля класс, фамилия, имя, отчество, специалист заполняются информацией из окон "Специалист", "Класс", "Фамилия", "Имя", "Отчество", а поля критерий и результат 
информацией из соответствующих полей временной таблицы, временная таблица очищается.

10.4 Вкладка для мониторинга.

Окно для проведения мониторинга открывается нажатием кнопки "Мониторинг", которую надо разместить в основном окне, правее кнопки "Перевод". Окно "Мониторинг" должно содержать несколько блоков из кнопок и окошек, блоки располагаются один под другим.
Верхний блок - выбор ученика и специалиста содержит два ряда кнопок и окон - в первом ряду расположить окно "Класс", кнопку "Найти", окно "Специалист" и кнопка "Выбор", ниже располагается ряд окон "Фамилия", "Имя", "Отчество", кнопки "Выгрузить в Excel" и "Очистить".
У окна "Класс" иметь значок для открытия всплывающего меню, в котором отражаются номера классов школы из таблицы "forms", после нажатия на номер нужного класса меню закрывается, а окно заполняется. Кнопка "Найти" должна иметь функционал всплывающего окна со списком выбранного класса, иметь возможность выбора конкретного ученика. После выбора ученика всплывающее меню закрывается, а поля Фамилия, Имя и Отчество заполняются.
Окно "Специалист" должно иметь значок для открытия выпадающего меню для выбора специалиста из таблицы "Специалисты".
После нажатия кнопки выбор информация о критериях и результатах оценки выбранного специалиста по выбранному ученику из таблицы "Анализ" должна отразиться в сформированную в блоке временную таблицу (в памяти, не в SQLite) с полями "Критерий" и "Результат", при этом полей "Результат" может быть несколько, в зависимости от количества периодов анализа.
Нажатие на кнопку "Выгрузить в Excel" должно сформировать файл в формате Excel, содержащий таблицу с информацией о критериях оценки и результатах по этим критериям. В шапке таблицы должны содержаться сведения о номере класса, о ФИО ученика и названии специалиста.
Кнопка "Очистить" обеспечивает очистку временной таблицы и всех окошек.
